# 任务目标
按步骤执行软件测试用例，并根据执行结果判断标准，按指定格式输出测试用例执行是否成功。
使用 Playwright MCP 提供的 **视觉截图功能（对应 API：page.screenshot()）**，为测试用例的**每一步操作完成后**生成页面视觉快照（最终搜索结果页必须生成），文件格式为 PNG，文件名为「当前时间戳_步骤名称」（最终结果页直接命名为当前时间戳）。


# 关键说明（必须严格遵守）
1.  页面快照 = 视觉像素截图（PNG 格式图片文件），**不是页面结构文档、不是 Markdown 文档、不是 DOM 序列化文本**，仅保留页面视觉呈现。
2.  Playwright MCP 中无单独的 browser_snapshot 函数，生成视觉截图请调用 `page.screenshot()` 方法，该方法是实现浏览器页面视觉快照的核心。
3.  文件名要求：
    - 步骤截图：格式为「{当前时间戳}_{步骤名称}」。
    - 必须通过 Playwright MCP 的 `page.evaluate()` 方法执行 JavaScript 代码生成时间戳，核心代码：`Math.floor(Date.now() / 1000)`（10 位秒级）。
    - 生成的时间戳需在截图前完成，且每个步骤的时间戳递增（避免文件名重复），例如步骤1用 `timestamp`，步骤2用 `timestamp+1`。
    - 禁止使用客户端传递的固定时间戳，必须由 MCP 侧实时生成。
    - 最终结果页截图：格式为「{当前时间戳}」。

4.  截图质量要求：全屏截图（包含完整页面内容，无截断），清晰可辨，确保能从截图中看到对应步骤的执行结果。

# 执行要求
- 先制定详细执行计划（包含：准备阶段、步骤执行阶段、截图阶段、断言阶段、结果整理阶段），再按计划执行。
- 控制浏览器、获取页面内容、生成截图，**必须通过 Playwright MCP 的 functioncall 函数调用实现**，禁止手动模拟。
- 每一步操作完成后，先验证界面是否符合预期（避免与测试用例严重不一致），再生成对应步骤截图；若界面不一致，立即判定测试失败，停止后续步骤，并记录失败原因。
- 若截图生成失败（如目录无权限、API 调用异常），直接判定测试用例执行失败，并在结果中注明截图失败原因。
